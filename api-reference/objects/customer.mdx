# Objeto customer

Representa as informações de um cliente no sistema PrimePix.

## Estrutura

```json
{
  "name": "João Silva",
  "document": "12345678900",
  "email": "joao@example.com",
  "phone": "+5511999999999"
}
```

## Propriedades

<ResponseField name="name" type="string">
  Nome completo do cliente.
</ResponseField>

<ResponseField name="document" type="string">
  CPF ou CNPJ do cliente (apenas números).
  - **CPF**: 11 dígitos
  - **CNPJ**: 14 dígitos
</ResponseField>

<ResponseField name="email" type="string">
  Endereço de e-mail do cliente.
</ResponseField>

<ResponseField name="phone" type="string">
  Número de telefone com código do país.
  
  Formato: `+{código_país}{DDD}{número}`
  
  Exemplo: `+5511999999999`
</ResponseField>

## Validações

### CPF

```javascript
// CPF deve ter 11 dígitos
"12345678900" // ✓ Válido
"123.456.789-00" // ✗ Inválido (contém formatação)
"1234567890" // ✗ Inválido (10 dígitos)
```

### CNPJ

```javascript
// CNPJ deve ter 14 dígitos
"12345678000190" // ✓ Válido
"12.345.678/0001-90" // ✗ Inválido (contém formatação)
```

### Email

```javascript
"joao@example.com" // ✓ Válido
"joao.silva@empresa.com.br" // ✓ Válido
"joao@" // ✗ Inválido
"@example.com" // ✗ Inválido
```

### Telefone

```javascript
"+5511999999999" // ✓ Válido (Brasil)
"+5521987654321" // ✓ Válido (Brasil)
"11999999999" // ✗ Inválido (falta código do país)
"+55 11 99999-9999" // ✗ Inválido (contém formatação)
```

## Exemplo de uso

### Criar transação com dados do cliente

```javascript Node.js
const response = await fetch('https://api.primepix.me/v1/pix/create', {
  method: 'POST',
  headers: {
    'X-API-Key': 'your_api_key',
    'X-API-Secret': 'your_api_secret',
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    amount: 100.00,
    description: 'Pagamento pedido #123',
    customer: {
      name: 'João Silva',
      document: '12345678900',
      email: 'joao@example.com',
      phone: '+5511999999999'
    }
  })
});

const { data } = await response.json();
console.log('Cliente:', data.customer);
```

```python Python
import requests

response = requests.post(
    'https://api.primepix.me/v1/pix/create',
    headers={
        'X-API-Key': 'your_api_key',
        'X-API-Secret': 'your_api_secret',
        'Content-Type': 'application/json'
    },
    json={
        'amount': 100.00,
        'description': 'Pagamento pedido #123',
        'customer': {
            'name': 'João Silva',
            'document': '12345678900',
            'email': 'joao@example.com',
            'phone': '+5511999999999'
        }
    }
)

data = response.json()['data']
print('Cliente:', data['customer'])
```

## Boas práticas

<AccordionGroup>
  <Accordion title="Sempre remova formatação de documentos">
    Envie CPF e CNPJ apenas com números, sem pontos, traços ou barras.
    
    ```javascript
    // ✓ Correto
    const document = cpf.replace(/\D/g, ''); // Remove tudo que não é número
    
    // ✗ Incorreto
    const document = "123.456.789-00";
    ```
  </Accordion>

  <Accordion title="Valide e-mails antes de enviar">
    Use uma biblioteca de validação ou regex para garantir que o e-mail está no formato correto.
    
    ```javascript
    const emailRegex = /^[^\s@]+@[^\s@]+\.[^\s@]+$/;
    if (!emailRegex.test(email)) {
      throw new Error('E-mail inválido');
    }
    ```
  </Accordion>

  <Accordion title="Inclua código do país no telefone">
    Sempre inclua o código do país (+55 para Brasil) no início do número.
    
    ```javascript
    // ✓ Correto
    const phone = '+55' + ddd + numero;
    
    // ✗ Incorreto
    const phone = ddd + numero;
    ```
  </Accordion>

  <Accordion title="Armazene dados do cliente">
    Guarde as informações do cliente para facilitar futuras transações e melhorar a experiência.
  </Accordion>
</AccordionGroup>

## Privacidade e segurança

<Warning>
  **Proteção de dados pessoais**
  
  As informações do cliente são dados pessoais protegidos pela LGPD (Lei Geral de Proteção de Dados). Certifique-se de:
  
  - Obter consentimento do cliente para armazenar seus dados
  - Armazenar dados de forma segura e criptografada
  - Não compartilhar dados com terceiros sem autorização
  - Permitir que clientes acessem, corrijam ou excluam seus dados
</Warning>
