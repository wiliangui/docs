# List transactions

`GET /api/v1/transactions`

Retrieves a paginated list of all transactions with optional filtering.

## Request

### Headers

| Header | Type | Required | Description |
|--------|------|----------|-------------|
| `X-API-Key` | string | Yes | Your API key |
| `X-API-Secret` | string | Yes | Your API secret |

### Query parameters

<ParamField query="page" type="integer" default="1">
  Page number for pagination.
</ParamField>

<ParamField query="limit" type="integer" default="20">
  Number of transactions per page (max 100).
</ParamField>

<ParamField query="status" type="string">
  Filter by transaction status. Possible values:
  - `pending`
  - `completed`
  - `failed`
  - `cancelled`
</ParamField>

<ParamField query="type" type="string">
  Filter by transaction type. Possible values:
  - `payment` - PIX payments received
  - `withdrawal` - Withdrawals made
  - `fee` - Transaction fees
  - `refund` - Refunds issued
</ParamField>

<ParamField query="startDate" type="string">
  Filter transactions from this date (ISO 8601 format: `2024-01-01T00:00:00Z`).
</ParamField>

<ParamField query="endDate" type="string">
  Filter transactions until this date (ISO 8601 format: `2024-01-31T23:59:59Z`).
</ParamField>

## Response

<ResponseField name="success" type="boolean">
  Indicates if the request was successful.
</ResponseField>

<ResponseField name="data" type="array">
  Array of transaction objects.
  
  <Expandable title="transaction properties">
    <ResponseField name="id" type="integer">
      Unique transaction identifier.
    </ResponseField>
    
    <ResponseField name="externalId" type="string">
      Your external reference ID (if provided).
    </ResponseField>
    
    <ResponseField name="type" type="string">
      Transaction type (payment, withdrawal, fee, refund).
    </ResponseField>
    
    <ResponseField name="amount" type="number">
      Transaction amount.
    </ResponseField>
    
    <ResponseField name="fee" type="number">
      Fee charged for this transaction.
    </ResponseField>
    
    <ResponseField name="netAmount" type="number">
      Net amount (amount - fee for payments, -amount for withdrawals).
    </ResponseField>
    
    <ResponseField name="status" type="string">
      Current transaction status.
    </ResponseField>
    
    <ResponseField name="description" type="string">
      Transaction description.
    </ResponseField>
    
    <ResponseField name="createdAt" type="string">
      ISO 8601 timestamp when transaction was created.
    </ResponseField>
    
    <ResponseField name="completedAt" type="string">
      ISO 8601 timestamp when transaction was completed (if applicable).
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="pagination" type="object">
  Pagination metadata.
  
  <Expandable title="pagination properties">
    <ResponseField name="page" type="integer">
      Current page number.
    </ResponseField>
    
    <ResponseField name="limit" type="integer">
      Number of items per page.
    </ResponseField>
    
    <ResponseField name="total" type="integer">
      Total number of transactions.
    </ResponseField>
    
    <ResponseField name="totalPages" type="integer">
      Total number of pages.
    </ResponseField>
  </Expandable>
</ResponseField>

## Filtering examples

### Get all completed payments from last month

```javascript Node.js
const startDate = new Date('2024-01-01T00:00:00Z').toISOString();
const endDate = new Date('2024-01-31T23:59:59Z').toISOString();

const params = new URLSearchParams({
  type: 'payment',
  status: 'completed',
  startDate,
  endDate,
  limit: '100'
});

const response = await fetch(
  `https://api.primepix.me/v1/transactions?${params}`,
  {
    headers: {
      'X-API-Key': process.env.PRIMEPIX_API_KEY,
      'X-API-Secret': process.env.PRIMEPIX_API_SECRET
    }
  }
);

const { data: transactions } = await response.json();
```

### Calculate total revenue

```javascript Node.js
async function calculateRevenue(startDate, endDate) {
  let page = 1;
  let totalRevenue = 0;
  let hasMore = true;
  
  while (hasMore) {
    const params = new URLSearchParams({
      type: 'payment',
      status: 'completed',
      startDate,
      endDate,
      page: page.toString(),
      limit: '100'
    });
    
    const response = await fetch(
      `https://api.primepix.me/v1/transactions?${params}`,
      {
        headers: {
          'X-API-Key': process.env.PRIMEPIX_API_KEY,
          'X-API-Secret': process.env.PRIMEPIX_API_SECRET
        }
      }
    );
    
    const { data: transactions, pagination } = await response.json();
    
    totalRevenue += transactions.reduce(
      (sum, tx) => sum + tx.netAmount,
      0
    );
    
    hasMore = page < pagination.totalPages;
    page++;
  }
  
  return totalRevenue;
}
```

## Pagination best practices

<AccordionGroup>
  <Accordion title="Use appropriate page sizes">
    Request only the data you need. Smaller page sizes (20-50) are better for UI display, while larger sizes (100) are better for data processing.
  </Accordion>

  <Accordion title="Cache results when possible">
    If you're displaying historical data that doesn't change, cache the results to reduce API calls.
  </Accordion>

  <Accordion title="Handle pagination metadata">
    Always check the `totalPages` field to know when to stop paginating.
  </Accordion>

  <Accordion title="Use date filters for large datasets">
    When dealing with many transactions, use `startDate` and `endDate` to narrow down results.
  </Accordion>
</AccordionGroup>
