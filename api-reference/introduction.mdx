# Introdução

Bem-vindo à documentação da API PrimePix!

Nossa API segue os padrões REST, com respostas em formato JSON e autenticação via API Key.

## URL base

```
https://api.primepix.me/v1
```

## Autenticação

Para autenticar com nossa API, você deve passar sua **API Key** e **API Secret** nos headers de todas as requisições:

```bash
X-API-Key: sua_api_key
X-API-Secret: sua_api_secret
```

Você pode encontrar suas chaves acessando o menu [Configurações → Credenciais de API](https://app.primepix.me/settings/credentials) no dashboard.

## Exemplo de requisição

<CodeGroup>

```bash cURL
curl -X GET https://api.primepix.me/v1/balance \
  -H "X-API-Key: sua_api_key" \
  -H "X-API-Secret: sua_api_secret"
```

```javascript Node.js
const response = await fetch('https://api.primepix.me/v1/balance', {
  headers: {
    'X-API-Key': 'sua_api_key',
    'X-API-Secret': 'sua_api_secret'
  }
});

const data = await response.json();
console.log(data);
```

```python Python
import requests

response = requests.get(
    'https://api.primepix.me/v1/balance',
    headers={
        'X-API-Key': 'sua_api_key',
        'X-API-Secret': 'sua_api_secret'
    }
)

data = response.json()
print(data)
```

```php PHP
<?php
$ch = curl_init('https://api.primepix.me/v1/balance');

curl_setopt($ch, CURLOPT_RETURNTRANSFER, true);
curl_setopt($ch, CURLOPT_HTTPHEADER, [
    'X-API-Key: sua_api_key',
    'X-API-Secret: sua_api_secret'
]);

$response = curl_exec($ch);
curl_close($ch);

$data = json_decode($response, true);
print_r($data);
?>
```

</CodeGroup>

## Formato das respostas

Todas as respostas seguem um formato padrão JSON:

### Resposta de sucesso

```json
{
  "success": true,
  "data": {
    // Dados da resposta
  }
}
```

### Resposta de erro

```json
{
  "success": false,
  "error": "Mensagem de erro",
  "code": "ERROR_CODE",
  "details": {
    // Detalhes adicionais do erro
  }
}
```

## Códigos de status HTTP

| Código | Descrição |
|--------|-----------|
| `200` | Sucesso - Requisição processada com sucesso |
| `400` | Bad Request - Parâmetros inválidos |
| `401` | Unauthorized - Credenciais inválidas ou ausentes |
| `403` | Forbidden - Sem permissão para acessar o recurso |
| `404` | Not Found - Recurso não encontrado |
| `422` | Unprocessable Entity - Erro de validação |
| `429` | Too Many Requests - Limite de requisições excedido |
| `500` | Internal Server Error - Erro no servidor |
| `503` | Service Unavailable - Serviço temporariamente indisponível |

## Rate limiting

A API possui limites de requisições para garantir uso justo:

- **100 requisições por minuto** por API key
- **1000 requisições por hora** por API key

Quando o limite é excedido, você receberá um erro `429 Too Many Requests`:

```json
{
  "success": false,
  "error": "Rate limit exceeded",
  "code": "RATE_LIMIT_EXCEEDED",
  "details": {
    "retryAfter": 60
  }
}
```

O header `Retry-After` indica quantos segundos você deve aguardar antes de tentar novamente.

## Paginação

Endpoints que retornam listas suportam paginação via query parameters:

```bash
GET /v1/transactions?page=1&limit=20
```

### Parâmetros

- `page` - Número da página (padrão: 1)
- `limit` - Itens por página (padrão: 20, máximo: 100)

### Resposta com paginação

```json
{
  "success": true,
  "data": [...],
  "pagination": {
    "page": 1,
    "limit": 20,
    "total": 150,
    "totalPages": 8
  }
}
```
