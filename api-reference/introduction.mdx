# API Reference

The PrimePix API is organized around REST principles. It uses standard HTTP methods, authentication, and response codes.

## Base URL

```
https://api.primepix.me/v1
```

## Authentication

All API requests require authentication using API Key and API Secret headers:

```bash
X-API-Key: your_api_key
X-API-Secret: your_api_secret
```

Learn more about [authentication](/authentication).

## Request format

All POST requests should include a `Content-Type: application/json` header and a JSON body.

```bash
curl -X POST https://api.primepix.me/v1/pix/create \
  -H "X-API-Key: your_api_key" \
  -H "X-API-Secret: your_api_secret" \
  -H "Content-Type: application/json" \
  -d '{"amount": 100.00}'
```

## Response format

All responses are returned in JSON format with a `success` field indicating the result:

### Success response

```json
{
  "success": true,
  "data": {
    // Response data
  }
}
```

### Error response

```json
{
  "success": false,
  "error": "Error message",
  "code": "ERROR_CODE"
}
```

## Rate limiting

The API has rate limits to ensure fair usage:

- **100 requests per minute** per API key
- **1000 requests per hour** per API key

When you exceed the rate limit, you'll receive a `429 Too Many Requests` response.

## Pagination

List endpoints support pagination using `page` and `limit` query parameters:

```bash
GET /api/v1/transactions?page=1&limit=20
```

Response includes pagination metadata:

```json
{
  "success": true,
  "data": [...],
  "pagination": {
    "page": 1,
    "limit": 20,
    "total": 150,
    "totalPages": 8
  }
}
```

## Endpoints

### PIX payments

- [POST /pix/create](/api-reference/pix/create) - Create a PIX charge
- [GET /pix/status/{id}](/api-reference/pix/status) - Get payment status

### Account management

- [GET /balance](/api-reference/account/balance) - Get account balance
- [GET /transactions](/api-reference/account/transactions) - List transactions
- [POST /withdraw](/api-reference/account/withdraw) - Request withdrawal
