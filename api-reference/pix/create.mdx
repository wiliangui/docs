# Create PIX charge

`POST /api/v1/pix/create`

Generates a PIX QR Code for receiving payments from customers.

## Request

### Headers

| Header | Type | Required | Description |
|--------|------|----------|-------------|
| `X-API-Key` | string | Yes | Your API key |
| `X-API-Secret` | string | Yes | Your API secret |
| `Content-Type` | string | Yes | Must be `application/json` |

### Body parameters

<ParamField body="amount" type="number" required>
  The charge amount in Brazilian Reais (BRL).
</ParamField>

<ParamField body="externalId" type="string">
  Your internal reference ID for this transaction. Must be unique.
</ParamField>

<ParamField body="description" type="string">
  Description of the charge that will be displayed to the customer.
</ParamField>

<ParamField body="customer" type="object">
  Customer information for the transaction.
  
  <Expandable title="customer properties">
    <ParamField body="name" type="string">
      Customer's full name.
    </ParamField>
    
    <ParamField body="document" type="string">
      Customer's CPF or CNPJ (numbers only).
    </ParamField>
    
    <ParamField body="email" type="string">
      Customer's email address.
    </ParamField>
    
    <ParamField body="phone" type="string">
      Customer's phone number with country code.
    </ParamField>
  </Expandable>
</ParamField>

## Response

<ResponseField name="success" type="boolean">
  Indicates if the request was successful.
</ResponseField>

<ResponseField name="data" type="object">
  The charge data.
  
  <Expandable title="data properties">
    <ResponseField name="transactionId" type="integer">
      Unique transaction identifier.
    </ResponseField>
    
    <ResponseField name="qrCode" type="string">
      PIX code string for copy/paste payment.
    </ResponseField>
    
    <ResponseField name="qrCodeBase64" type="string">
      Base64-encoded QR code image.
    </ResponseField>
    
    <ResponseField name="amount" type="number">
      The charge amount.
    </ResponseField>
    
    <ResponseField name="fee" type="number">
      Transaction fee charged.
    </ResponseField>
    
    <ResponseField name="netAmount" type="number">
      Net amount you'll receive (amount - fee).
    </ResponseField>
    
    <ResponseField name="status" type="string">
      Current transaction status. Initially `pending`.
    </ResponseField>
  </Expandable>
</ResponseField>

## Status codes

| Status | Description |
|--------|-------------|
| `200` | Charge created successfully |
| `400` | Invalid request parameters |
| `401` | Invalid or missing authentication |
| `422` | Validation error |
| `500` | Internal server error |

## Common errors

### Missing amount

```json
{
  "success": false,
  "error": "Validation failed",
  "code": "VALIDATION_ERROR",
  "details": {
    "amount": "Amount is required"
  }
}
```
